## 1. ゴール定義: 10月の「形」とは何か

**仮定**: 10月 = 中間発表相当のチェックポイント（修論本体は先）。そこまでに揃えるべき「形」を3点セットで定義する:

| # | 成果物 | 完成の定義 |
|---|---|---|
| A | **定量実証**: 「シンプル報酬でも co-design はタスクに適した形態を識別できる」 | 3タスク（Pusher / Reach / Target-Pusher）× 2 seed の形態比較表（L11・gear・tip_reach）で、タスク別の形態差が seed を跨いで再現すること |
| B | **動くシステム**: スケッチ → 形態妥当性検証パイプライン | 手描き（or Gemini 生成）スケッチを入力すると、人手ゼロで co-design 学習が走り「この形態でこのタスクは筋が良いか」の判定が出る E2E デモ |
| C | **論文骨子**: contribution 3本柱のドラフト | ①固定根本アームへの適用と環境是正の記録 ②複数タスク形態比較の実証 ③スケッチ→co-design パイプライン、を実験系譜.md ベースで文章化 |

A が研究の核心、B がビジョンの証明、C が発表物。**A なしの B はデモ止まり、B なしの A は再現実験止まり**なので両方を狙う。

## 5. 実験予算（GPU 1枚での成立性）

実測: 5〜6本並走で 7〜8分/ep → **1000ep ≈ 5日、200ep ≈ 1日**。

| 期間 | 走らせる run | 本数 |
|---|---|---|
| 〜7/13 | ✅ 消化済み: I1/I2・TP1 完走、L 系 4本 | 6 |
| 7/13〜7/20 | L2/L2_s2（完走）+ **TP2/TP2_s2（1000ep、最後の大物）** | 4 |
| 7/20〜7/末 | 完走処理・G1/A 確定のみ（新規 run なし = バッファ） | 0〜1 |
| 8月 | tripo 本格 1〜2 本 + 高速判定の短期 run 多数（200ep 以下） | 小口多数 |
| 9月 | マトリクス短期 run（warm-start 200ep × 4〜6） | 小口多数 |

**成立する**。ただし前提は (a) 打ち切り基準の運用（plateau 放置しない）、(b) 1000ep の大物は 7月中に集中させ、8月以降は warm-start 短期 run 主体に切り替えること。長期 run の追加判断は都度「その 5 日で主張のどこが強くなるか」で審査する。

## 6. リスクと対策

| リスク | 兆候 | 対策 |
|---|---|---|
| L1/L2 で形態差が出ない | G1 ゲートで判明 | 分岐先を事前定義済み（gear 範囲・bounds 拡大で1回だけ再実験）。それでもダメなら「識別できない条件の特定」自体を negative result として主張②を再構成 |
| scale=1 長期学習の崩壊（J1 型） | plateau 後の急落 | 打ち切り基準の機械的適用 + best.p 採用（崩壊前の checkpoint で評価） |
| Tripo3D の出力揺らぎ（色・形状） | 新 GLB で検出失敗 | 色自動キャリブレーション（Phase 4 冒頭）+ chain モードフォールバック |
| seed 追加が間に合わない | 8月後半に GPU 逼迫 | 主張②は 2 seed で主張し、3 seed 目は「あれば強化」扱いに割り切る |
| 10月の「形」の定義ずれ | — | 本計画の §1 を指導教員と早期にすり合わせる（**7月中推奨**） |
