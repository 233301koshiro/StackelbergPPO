## Q1: 「Pusher に長い腕が良いのは自明では？ 原論文の貢献の主要部は関節を増やせること（トポロジー探索）で、リンク長だけの最適化なら既存のもっと簡単な手法があるのでは？」

（出典: ユーザー自身の疑問、2026-07-15。トポロジー固定 = fix_skeleton=true を本線とする本研究の急所を突く質問）

### 認める部分（正直に認めるのが最善）

- **「Pusher に長い腕」は自明**。先端速度 = 角速度 × 総リンク長という力学から紙の上で導ける。L 系 Pusher のリーチが伸びたこと自体に発見の価値はない
- **属性（連続パラメータ）のみの形態最適化は既存領域**。Ha 2019（design パラメータの RL 同時最適化）、Schaff et al. 2019（設計分布と方策の同時学習）、進化計算で候補ごとに制御を学習する系譜など [要文献確認]。「パラメータ数個を最適化できる」ことを貢献に据えたら本研究は弱い
- 原論文の新規性にトポロジー探索が含まれるのも事実（本研究はそれを意図的に使わない）

### データによる反論 — 属性のみの世界でも非自明だった実測結果

1. **gear の分岐は事前予測を外した**。構造分析の当初予測は「トルクコストほぼ無料なら両タスクとも gear 上限」。Reach の下限 20 収束は**反証として発見**され、その機構（決定論の到達精度は gear に依存しない・確率的ロールアウトの誤着地を 10/10 vs 5/10 で抑制する頑健性）は n=10 の検証を要した。解析で自明に導ける範囲の外にある
2. **Reach の内部解**（総リーチ 0.9〜1.1、探索上限 1.44 に非接触、2 seed）は「長いほど良い」の否定形 = 「必要十分で止まる」ことの実証
3. **同じ Pusher 内でも自明でない構造**: 中庸型 102.2 vs 長腕型 145.4（1.4 倍、n=2）、K2 の「収束先は同じなのに中盤の学習動力学だけが壊れる」現象 — 力学の紙計算からは出ない
4. **Target-Pusher は形態がほぼ Pusher と同じ（高 gear・中リーチ）なのに行動で分化**（押して止める vs 吹き飛ばす）。つまり「形態が自明かどうか」と独立に、**制御の習熟込みの評価でなければタスク適性は判定できない** — 形態だけ見たら TP と Pusher は区別不能。これは Stackelberg の return 継承（Leader が Follower の実応答で評価される）を判定に使う本研究の設計の直接の根拠になる

### 貢献の位置づけ（防御線）

本研究は co-design を「より良い形を設計する道具」ではなく「**人間が描いた形の筋の良さを、制御の習熟込みで判定する検証器**」として使う（B 軸、論文骨子）。この用途では:

- **トポロジー固定は制約ではなく仕様** — スケッチが決めた構造を尊重して評価するのが目的で、「関節を増やせる」能力はむしろ使ってはいけない
- 問われる性質は「最適解を見つける力」ではなく「**判定の妥当性**（タスクの要求を正しく反映するか = C2、原論文はタスク間比較をしていない空白）と**判定の速度**（転用設計則・ep100 較正 = C3）」
- 「リンク長最適化ならもっと良い手法がある」は真でも、**判定器としての妥当性検証・スケッチ入力の E2E システム・タスク間比較は先行研究にない**

### 裏付け実験（計画済み）

**属性成分の分解 ablation**（M 系、07-15 起動済み）: fix_skeleton=true のまま (a) M1 = 長さのみ可変（gear 凍結）(b) M2b = gear のみ可変（長さ凍結 + 非接触スタート）(c) 両方 = L2 実績、の3条件 × Pusher 200ep。

**起動前に確定した部分回答**: gear のみ・素の条件（M2）は全エピソードが初期接触ペナルティに張り付き学習不能だった。デフォルト形態は対象に最初から貫通しており、そこから脱出できるのはリンク長の変更だけ — すなわち**リンク長の可変性は「より良い形を探す」以前に、この環境でタスクを成立させるための必要条件**である。「長い腕が良いのは自明」という批判に対し、「自明な方向に動けること自体が学習成立の前提であり、それを踏まえた上で gear・行動の非自明な分化が乗る」という2層の答えになる。（M1/M2b の定量値は完走後に追記）

### 想定される再反論への備え

- 「gear も自明では（強いモーターが良いに決まっている）」→ Reach は**下限**を選んだ（2 seed + 旧世代2 run の計4本）。「強いほど良い」はタスク依存で、その依存の仕方（確率的頑健性経由）が非自明
- 「n=2 で言えるのか」→ 認める。タスク間分離は4軸すべてで seed 跨ぎ一致だが、主張は「2 seed の範囲で頑健」に統一している（修論の限界節）
- 「判定器なら比較対象は他の判定手法（IK 解析・幾何チェック）では」→ 正しい比較軸。幾何チェック（リーチ判定等）は「届くか」しか答えられず、「押して止められるか」のような制御依存の適性は学習ベースでないと判定できない、が答え。TP の形態同一・行動分化がその実例
