## 4. 落とし穴（新セッションが踏みがちなもの）

- **Bug 8**: pgrep/pkill -f は自分自身にマッチする。ブラケット記法（`pgrep -f "foo_[b]ar"`）を使い、kill 系は単独の Bash 呼び出しで実行。監視は log-grep 方式を優先
- **学習の起動形式**: `nohup env USE_CHOREONOID=1 OMP_NUM_THREADS=1 /choreonoid_ws/install/bin/choreonoid --no-window --python scripts/choreonoid_train.py cfg=... hydra.run.dir=single_run/<name> > .../stdout.log 2>&1 &`（bash -c で包まない）
- **eval スクリプト**: `timeout -s KILL` 必須（Qt が例外を握って hang する）、`torch.set_default_dtype(torch.float64)`、保存 config の restore_dir は OmegaConf.update で上書き
- **判定指標**: 形態の良し悪しは決定論 eval 単発で判定しない（確率的評価と優劣が逆転する実測あり、形態比較.md「L1/L1_s2 完走解析」）。学習曲線の ep75/100 順位法を使う
- **md の信頼度**: 一次データ（log/overrides）> 実験系譜・形態比較（監査済み）> 進捗.md の古い節。取り消し線・「⚠️訂正」の後だけ信じる
- ユーザーの文体好み: md は平易な日本語（ゼロスタート・頭打ち等）+ 用語は実験系譜.md 冒頭の辞典に集約。コミットは内容別に分ける。修論ドラフト本文のみ学術文体（だ・である）
