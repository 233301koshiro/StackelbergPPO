## 推奨読み順

> **研究目標**: 「シンプルな報酬で、Stackelberg PPO の co-design はタスクに適した形態を正しく識別できるか」を検証し、スケッチ → 形態の妥当性を素早く確認できるシステムの基盤を構築する。現在のスコープはアームロボット（固定根本・操作タスク）に限定。詳細は [`研究応用/研究方針.md`](研究応用/研究方針.md) 参照。

### Phase 1: 全体像を掴む（約 20 分、必須）
1. [README.md](../README.md) — 何のプロジェクトか・どう動かすか
2. [リポジトリ説明/システム概要.md](リポジトリ説明/システム概要.md) — Stackelberg PPO の仕組み（リーダー/フォロワー・3フェーズ・ログ指標の意味）

### Phase 2: 現在の状態を把握する（約 15 分、必須）
3. **[進捗.md](進捗.md)の「TL;DR」と「現在の状態」テーブルだけ読む** — 今どこまで来ているか・重大バグ（関節 damping 無視）修正済みかの確認
4. [デバッグ戦記.md](デバッグ戦記.md) — 発見したバグの一覧・**次に詰まったときの調査手順**（必ず読む）

### Phase 3: 実装を理解する（約 30 分、実装するとき）
5. [リポジトリ説明/コードベース概要.md](リポジトリ説明/コードベース概要.md) — コード構成と1エポックの処理フロー
6. [移行レポート.md](../移行レポート.md) — なぜ Choreonoid を使うか・起動方法・既知の制限

### Phase 4: 研究固有の課題（深掘り時に）
7. [研究応用/研究方針.md](研究応用/研究方針.md) — **研究ゴール・スコープ・タスクロードマップ・論文との関係**（まずここを読む）
8. [研究応用/修士研究計画_2026-07.md](研究応用/修士研究計画_2026-07.md) — **10月までの研究計画**（「形」の定義・フェーズ・ゲート・実験予算・リスク）
9. [研究応用/タスク設計と報酬関数.md](研究応用/タスク設計と報酬関数.md) — 報酬設計の試行錯誤と教訓（複雑化がなぜ失敗したか）
10. [研究応用/実験系譜.md](研究応用/実験系譜.md) — **対照実験・知見・次の仮説の連鎖を一望**（研究ストーリーの背骨。論文の実験構成にもここから）
11. [研究応用/形態比較.md](研究応用/形態比較.md) — **タスク別最適形態の実測記録**（gear 20 vs 400・前腕長分岐など研究の核心データ）
12. [研究応用/方針レビュー_2026-07-08.md](研究応用/方針レビュー_2026-07-08.md) — 外部レビュー意見（比較設計の交絡・target_pusher 評価・推奨アクション・採否）
13. [進捗.md](進捗.md) — 実験ログ・バグ修正履歴（日報メモ的位置づけ）

### Phase 5: スケッチ→学習パイプライン（7月タスクに着手するとき）
9. [研究応用/メッシュ分割.md](研究応用/メッシュ分割.md) — 3D メッシュをロボットパーツに分割する手法の比較
10. [研究応用/メッシュXMLパイプライン.md](研究応用/メッシュXMLパイプライン.md) — メッシュ → MuJoCo XML 変換の設計方針（爆発問題の対策）・実証済みフルパイプライン（§8）
11. [研究応用/Tripo3D_Choreonoid可視化.md](研究応用/Tripo3D_Choreonoid可視化.md) — Gemini → Tripo3D → Choreonoid 可視化の手順と知見（Y-up変換・マゼンタマーカー）
12. [研究応用/トポロジー固定最適化.md](研究応用/トポロジー固定最適化.md) — 初期形状の指定・トポロジー固定で属性値のみ最適化する方法
13. [タスク計画.md](タスク計画.md) — **現在〜近期の実験・実装タスク**（L 系判定・TP 本走・Phase 3 手順）
14. [ロードマップ.md](../ロードマップ.md) — 6〜7月実装タスク記録（**凍結済み**）
