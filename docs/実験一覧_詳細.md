## データ解釈ガイド

### 列の説明

- **Run 名**: ディレクトリ名（`single_run/` 配下）
- **Task**: 評価タスク（Pusher, Reach, Target-Pusher 等）
- **cfg**: hydra config キー（`cfg=pusher` など）
- **Seed**: 乱数シード（N/A = config に明示されていない、または 1=seed=1 override）
- **fix_skeleton**: トポロジー固定フラグ（true=固定、default=デフォルト設定）
- **best_exec_R_eps**: Execution phase での最高報酬（ログから抽出）
  - Pusher では `fwd_cube` が正の値（速度）
  - Reach では `reach_dist` が負の値（距離のマイナス、つまり小さい = 近い）
  - Target-Pusher では PBRS 報酬（小さい値）
- **best_epoch**: その最高報酬を達成した epoch
- **final_epoch**: 学習が最後まで到達した epoch（200, 999 が一般的な終端）
- **Checkpoints**: 保存されたモデルの数（checkpoint 数 ≈ 学習の進行度）

### 実験グループ分け

**Pusher タスク群** (rows 1–9):
- H1, I1, I2: baseline + 転用実験
- K1, K2: 転用粒度（制御 vs 形態）
- L2, L2_s2: 最終是正版 + seed validation
- fixmorph_s1, morph_frozen: 形態凍結 baseline

**Reach タスク群** (rows 10–15):
- G3, G4: 重み転用と ゼロスタート比較
- J1: Pusher→Reach 異タスク転用
- L1, L1_s2: 最終是正版（target 内側化） + seed validation
- codesign_s1 (=F7): 全 run 中唯一 fix_skeleton=false（トポロジー自由）。G2 との対照は事後解釈ペア（実験系譜 第3段）

**Target-Pusher タスク群** (rows 16–18):
- TP1, TP2, TP2_s2: 第 3 タスク検証（stop 誘導戦略）

## 注記

- **No log**: tripo_arm_smoke は学習済みの smoke test のため log が無い可能性
- **exec_R_eps**: Execution phase での episode return（policy が学習した形態で実行した報酬）
- **Checkpoint 数**: 0 は未実行、1–50 は短期実験、>50 は長期実験の目安
