## 知見: penetration インパルス退化戦略の全容（2026-06-24）

### 共通パターン（v2・v3・v4 すべてで発生）

ロボットが何もしなくても cube が動く / 高報酬を得る **Reward Hacking** が複数の形で発生した。

#### v2（速度報酬・前腕 penetration）

v2 eval 動画で「アーム静止・cube が飛ぶ」現象を確認。

**原因**: bone_offset 最適化でアームが斜め方向（body_1: x=+0.497,y=−0.498, body_11: x=+0.494,y=−0.498）に成長。
`joints=0` の初期状態で**前腕（elbow→tip の線分）が cube を横切っている**。

```
elbow:  (0.830, −0.018)   ←cube 左面 x=0.85 直前
tip:    (1.323, −0.517)

前腕が cube x:[0.85,1.15], y:[−0.15,+0.15] を横切るのは:
  t=0.04 → (0.851, −0.038)  ← cube 内部に入る
  t=0.27 → (0.963, −0.150)  ← cube 下端で出る
```

→ エピソード開始直後に物理エンジンが分離 impulse → cube 飛翔。先端ではなく**前腕中間部**が刺さっていた。

#### v3（目標座標報酬・先端 penetration 疑惑）

v3 eval 動画で「アーム静止・cube がジリジリ進む」を確認。アーム短縮後の先端が cube に届くかどうかは未確定だが、類似の退化が起きた可能性が高い。

#### v4（cube が最初から target にある）

`cube_x_offset=0.5` で cube 初期位置を x=1.5m に移動したが、`target_x=1.5` と一致してしまった。
アームは何もしなくても `reward_target = 1/(1+0) = 1.0/step` を得られるため、アームが縮んで静止する退化戦略に収束（epoch 50 で確認）。

### 累積的な修正

| version | 報酬 | 修正内容 | 問題 |
|---|---|---|---|
| v2 | 速度 | — | 前腕 penetration → cube 飛翔 |
| v3 | 目標 x=1.5 | — | penetration 系退化（停止） |
| v4 | 目標 x=1.5 + offset=0.5 | penetration 防止 | cube が最初から target（停止） |
| **v5** | **目標 x=2.0 + offset=0.5** | **target を cube 初期位置より遠くに設定** | — |

詳細: `docs/研究応用/タスク設計と報酬関数.md`

## 知見: 野球バット戦術と報酬関数の問題（2026-06-23）

### 何がわかったか

`eval_cnoid_visual.py` でアーム動作を可視化したところ、rrbot アームが**野球バット型**の戦術を学習していることが判明した。

**観察**: アームを振り上げて cube に一撃を与え、cube が高速で飛んでいく。「持続的に押す」動作ではない。

**根本原因 (現在の報酬関数)**:
```python
reward_fwd_cube = Δx / dt  # cube の x 方向速度に比例
```
速度ベースの報酬は「cube を最大速度で吹き飛ばすこと」を最適化する。固定根本 2 関節アームでは、先端速度を最大化する戦略は「エルボー ≈ 0°（直線伸ばし）でショルダーを高速回転させる」こと。これは野球バットのスイングと同じ運動学。

**さらに、形態最適化も絡む**:  
`reward_fwd_contact = 1/(1 + ||cube - arm_tip||)` により、アームリンク長が cube に届くよう伸長される。結果として「長くなったアームをバットのように振る」挙動が強化される。

**元のリポジトリでバット戦術が出なかった理由**:  
元実装はモバイルベース（自由関節付き）ロボット。アクションペナルティ + ベース移動コストにより「ゆっくり前進して押し続ける」戦術がバット戦術より高報酬になる。固定根本アームにはその誘因がない。

### どう対処するか

`design_opt/envs/pusher.py` に **目標座標ベース報酬**を追加（`use_target_reward: true` で有効化）:

```python
# 新報酬: cube が target_pos に近いほど高報酬
reward_fwd_cube = 1.0 / (1.0 + dist(cube, [target_x, target_y]))
```

- バット戦術で cube を目標を超えて飛ばすと距離が大きくなり報酬が下がる
- 目標手前で止まっても報酬が下がる → 目標付近に制御して維持する動作が最適
- `reward_specs.target_x`, `target_y` で目標座標を設定可能（デフォルト: x=1.5, y=0）
- `use_target_reward: false`（デフォルト）で従来の速度ベース報酬を維持（後方互換）
