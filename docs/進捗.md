## 方針変更（2026-06-19）

### MuJoCo vs Choreonoid 比較検証をやめる理由

当初 MuJoCo と Choreonoid の学習曲線・重みを比較して「環境移行後もリポジトリ元の精度を担保できている」ことを証明する予定だったが、以下の理由で中止：

1. **費用対効果**: Choreonoid でも報酬は増加しており「学習した結果できなかったことができている」は事実。MuJoCo が100点として Choreonoid が80点だとしても、その差分の原因調査に要する時間が見通せない。
2. **検証方法の限界**: 学習曲線の形が一致することはほぼない（物理エンジンの数値特性が異なるため）。動画比較が最も意味があるが MuJoCo の ETA が8日以上かかる。
3. **優先事項**: Choreonoid でも一応学習が動いているので、比較検証より先に一連のシステム（topology_to_xml.py 等）を作ることを優先する。

→ **Task 2（比較レポート）はタスクとして残すが今やるべきことではない**

### 今後の方針

- 学習は MuJoCo + Choreonoid 両方バックグラウンドで継続（放置）
- 一連のシステム構築（ROADMAP Task 3.5、4）を優先
- 比較検証は両方の学習が十分収束した段階で実施
