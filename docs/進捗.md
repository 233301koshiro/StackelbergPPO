## 現在実行中

### Choreonoid版 rrbot_arm 学習（fix_skeleton=true、epoch 23〜）

**状態**: バックグラウンド実行中（PID 191720）
**ログ**: `single_run/rrbot_arm_cnoid/stdout.log`
**チェックポイント保存先**: `single_run/rrbot_arm_cnoid/models/`

起動コマンド（**fix_skeleton=true が必須**）:
```bash
USE_CHOREONOID=1 OMP_NUM_THREADS=1 /choreonoid_ws/install/bin/choreonoid \
  --no-window --python scripts/choreonoid_train.py \
  cfg=pusher xml_name=rrbot_arm num_threads=4 enable_wandb=false \
  fix_skeleton=true \
  'hydra.run.dir=single_run/rrbot_arm_cnoid'
```

**epoch 進捗（抜粋）**:
| epoch | exec_R_eps | 備考 |
|---|---|---|
| 0 | 〜0 | スケルトン変換フェーズ |
| 1 | -1285 | 収束前の一時発散 |
| 15〜16 | 負値 | 同様の一時発散 |
| 20 | +3607 | 回復 |
| 21 | +1775 | oscillation |
| 22 | +2578 | 回復傾向 |
| 23 | +2624 | 安定回復中 |

**重要**: `fix_skeleton=false`（デフォルト）だと関節が 3→4→5→6 と増加し qpos が 1e165〜1e300 に発散 → RunningNorm 破損 → 全観測が NaN → クラッシュ。必ず `fix_skeleton=true` で起動すること。

**報酬 oscillation が発散した場合の推奨ハイパーパラメータ変更**（現在は様子見）:
| パラメータ | 現在値 | 推奨値 | 理由 |
|---|---|---|---|
| `num_optim_epoch` | 10 | 5 | 240更新/epoch → 120 に削減（固定topology向け） |
| `policy_lr` | 5e-5 | 3e-5 | 安定化 |
| `max_grad_norm` | 5 | 2 | 勾配クリッピング強化 |
| `clip_epsilon` | 0.2 | 0.1 | PPO 更新幅を制限 |
| `skel_transform_nsteps` | 5 | 1 | fix_skeleton 時は 1 ステップで十分 |
